Git

  • Git status: checks what branch we are currently on.
  • Git checkout -b branch/name: creates a new branch and switches to it
  • Git process to start a new working branch after navigating to correct directory:
    1. git checkout main
    2. git checkout -b feature/branch-name - creates a new branch with specified name
    3. git status - confirming that working on newly created branch
    4. code . - opens VS code program with new working branch
  • Git process after work is completed on new branch:
    1. Ensure on correct branch from the command line
    2. git add -A - adds/commits all files in the working branch/directory to the staging area of Git
    3. git commit -m "description of changes"- commits them to the repository while the -m flag associates a brief message with the commit
    4. git pull origin main - a check to ensure branches are in sync
    5. git push origin branch-name - pushes changes to GitHub
  • GitHub process after pushing changes from GitBash:
    1. Switch branch to the one just added
    2. Click pull requests tab
    3. Click "New Pull Request"
    4. Compare added branch with main branch to view changed items
    5. Click "Create pull request"
    6. Add a description of the changes made in this branch and click "Create pull request" again
    7. Click "Merge pull request" and again to confirm
  • Git process to update the local repository with the newly merged feature:
    1. git checkout main
    2. git pull origin main
